# Task Manager

Разработать клиент–серверное веб-приложение для управления задачами по методологии Kanban
с поддержкой ролей пользователей и чата внутри задач.

## Стек технологий
- Backend: FastAPI, SQLite
- Frontend: React 18
- Auth: JWT

### Запуск backend
```powershell
cd backend
uvicorn app.main:app --reload
```
### Запуск frontend

```powershell
cd frontend
npm start
```

## проделанной работе

### Что реализовано на текущем этапе

#### Backend (FastAPI)

* Реализовано REST API на базе **FastAPI**
* Настроена **JWT-аутентификация**:

  * регистрация и авторизация пользователей;
  * защита эндпоинтов
* Реализована **ролевая модель доступа**:

  * администратор;
  * руководитель;
  * менеджер;
  * сотрудник
* Спроектирована и реализована база данных **SQLite**:

  * пользователи;
  * задачи;
  * статусы задач;
  * сообщения чата
* Реализовано управление задачами:

  * создание задачи;
  * получение списка задач;
  * изменение статуса задачи
* Реализован чат внутри задач:

  * получение сообщений;
  * добавление сообщений
* Реализована инициализация справочника статусов при запуске приложения
* Настроена валидация входных данных и обработка ошибок

  * используются HTTP-коды `400 / 401 / 403 / 422`
  * отсутствуют неконтролируемые ошибки `500`

---

### Зачем это было сделано

* Backend реализован как **независимый сервис**, готовый к подключению frontend-части
* Архитектура проекта:

  * соответствует принципам клиент–серверного взаимодействия;
  * легко расширяется без переписывания кода
* Реализованные механизмы (JWT, роли, справочники, чат):

  * отражают реальные требования к системам управления задачами
* Проект подготовлен для:

  * дальнейшей разработки интерфейса;
  * демонстрации работы системы;
  * использования в рамках курсовой работы

---

### Текущий статус проекта

* Backend полностью работоспособен и протестирован через Swagger и Postman
* API стабильно обрабатывает запросы и возвращает предсказуемые ответы
* Проект готов к переходу на этап разработки клиентской части (Frontend)
